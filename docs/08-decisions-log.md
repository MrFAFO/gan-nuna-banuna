# 08 - Decisions Log

# יומן החלטות - גן נונה בנונה

## החלטה 1 - סוג המוצר

נבחר לבנות אפליקציית מובייל אישית לכל גן, במקום אפליקציה כללית אחת לכל הגנים.

## החלטה 2 - גישת White Label

למרות שכל גן יקבל אפליקציה אישית, הקוד ייבנה כבסיס משותף שניתן לשכפל ולהתאים.

## החלטה 3 - הלקוח הראשון

האפליקציה הראשונה תהיה:

גן נונה בנונה

## החלטה 4 - טכנולוגיה

הטכנולוגיה המומלצת:

- Expo.
- React Native.
- TypeScript.
- Expo Router.

## החלטה 5 - לא להתחיל מקוד לפני UX ו-UI

הוחלט לא לקפוץ לפיתוח לפני שיש:

- אפיון מוצר.
- UX Flow.
- Screen Specs.
- UI Direction.
- תמונות UI למסכים מרכזיים.

## החלטה 6 - חידוש חוזה

הוחלט שחידוש חוזה יהיה חלק מה-MVP.

## החלטה 7 - חתימה דרך ספק חיצוני

לא נבנה חתימה משפטית בעצמנו.

החידוש יתבצע דרך ספק חתימה חיצוני, כדי לאפשר תהליך חתימה תקף, מקצועי ומתועד.

## החלטה 8 - חוזה לא מובנה באפליקציה

החוזה עצמו לא חייב להיות כתוב בתוך האפליקציה.

הגננת תעלה קובץ PDF קיים, והמערכת תשלח אותו לחתימה.

## החלטה 9 - כפתור כניסה להורים

במסך התחברות, כפתור כניסה להורים יהיה הכפתור הראשי והגדול.

כניסת צוות הגן תהיה משנית וקטנה יותר.

הסיבה:
ההורים הם קהל המשתמשים הרחב והמרכזי מבחינת חוויית לקוח.

## החלטה 10 - איור אישי של הגנן והגננת

יופיע איור אישי של הגנן והגננת במסך ההתחברות.

האיור צריך לשדר:

- צוות גן.
- חום.
- אמון.
- נגישות.
- סביבה חינוכית.

לא לשדר:

- פוסטר זוגי.
- אופנה.
- רומנטיקה מוגזמת.
- מערכת גנרית.

## החלטה 11 - התמונות יעברו ל-Claude Code כרפרנס

תמונות UI ישמשו כ-Visual Reference בלבד.

בפיתוח, המסך ייבנה מרכיבים אמיתיים:

- טקסט אמיתי.
- כפתורים אמיתיים.
- Layout אמיתי.
- Hero image כאסט.

## החלטה 12 - קבצי Markdown כמקור אמת

הוחלט לנהל את הפרויקט גם בקבצי Markdown מסודרים, ולא להסתמך רק על זיכרון השיחה.

הקבצים ישמשו:

- אותנו.
- את Claude Code.
- את Codex.
- את המשך הפיתוח.
